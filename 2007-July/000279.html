Summary: Wrong single select variables html generation for configlets

Details: the most of configlets have ("value", "Name") for each choice variable, but html code is using the Name as value, eg, for cfg_app_callfw.py:

<option value="Call Forwarding if Busy">Call Forwarding if Busy</option>

<option value="Call Forwarding if Timeout/Unavailable">Call Forwarding if Timeout/Unavailable</option>

It should be fixed in ConfigHelper.py

Follow-Ups:

Date: 2007-Jul-25 10:54
By: alerios

Comment:
Also, when the variables name doesn't use the _() function, the value field is ignored, and numbers are used instead.

options=[('fxs_ls', _('loopstart')), ('fxs_ks', _('kewlstart'))]),

**html without _():
<option selected="selected" value="0">loopstart</option>
<option value="1">kewlstart</option>
---
options=[('fxs_ls', 'loopstart'), ('fxs_ks', 'kewlstart')]),

**html with _():
<option selected="selected" value="fxs_ls">loopstart</option>
<option value="fxs_ks">kewlstart</option>
--
options=[('fxs_ls', _('loopstart')), ('fxs_ks', _('kewlstart'))]),
